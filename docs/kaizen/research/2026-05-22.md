## External Scan

### What's moving this week

The upstream-unlocks-our-backlog pattern paid off twice. **Strands v1.41.0** lands `cache_tools_ttl` (PR #2232) — previously SDK-managed cache points were hardcoded to 5m TTL, so 1h caching was structurally impossible because Bedrock requires non-increasing TTLs across `toolConfig → system → messages`. Our issue #269 ("Enable Bedrock prompt caching, `CacheConfig strategy=auto`") has been open since May 9; v1.41 is exactly the precondition it needed. The same release also bumps `starlette` into the 1.x major line (PR #2297) — a transitive-conflict check before the pin moves. Separately, **`bedrock-agentcore` jumped two releases** (1.10.0 May 19, 1.11.0 May 22) and the SDK issue tracker surfaced a genuinely scary one: **#482**, a blocking `queue.Queue.put()` with no timeout that freezes the runtime worker thread when an SSE consumer stops iterating — production-confirmed at a 78s stall on Lambda-rotation reconnect. We run exactly that architecture.

The **MCP spec locked its next release-candidate scope** (22 SEPs, RC for the 2026-06-30/07-28 revision). For us the consequential piece is MCP Apps graduating with **declare-templates-ahead-of-time**: tools advertise their `ui://` templates on the tool list so hosts can prefetch/cache/security-review *before* any tool runs. Our host-renderer (shipped beta.27) inlines App HTML at `ui_resource` SSE-emit time, *after* `tool_result` — the RC's model moves that fetch earlier. Not a break today, but a divergence to track before it stabilizes. The reference repo had a quiet week (7 commits, 4 relevant); the standout is an Opus 4.7 adoption commit that **omits `temperature` for any `opus-4-7` model ID** because extended-thinking models reject it — directly relevant since we just shipped Opus 4.7 adaptive thinking in beta.27 (#331). Frontier side: Google I/O dropped Gemini 3.5 Flash (1M context, agent-tuned, not on Bedrock yet) and Anthropic acquired Stainless and is winding down its hosted SDK generator.

- **Change**: (a) audit the `starlette` 1.x transitive bump against `fastapi==0.136.1` and the backend server stack before moving the pin; (b) bump `strands-agents` 1.40.0 → 1.41.0; (c) set `cache_tools_ttl` (and `CacheConfig.ttl`) so cache points across `toolConfig → system → messages` carry a 1h TTL — the non-increasing-TTL constraint is why this was structurally impossible on 1.40; (d) verify the admin "Cache Savings" card reflects the longer-lived cache hits.

### 2. Defensive guard against SDK #482 SSE-disconnect runtime deadlock
- **Source**: AgentCore SDK issue #482 (https://github.com/aws/bedrock-agentcore-sdk-python/issues/482)
- **Surface area**: `backend/src/apis/inference_api/` streaming worker path (the AgentCore Runtime `/invocations` SSE handler)
- **Change**: `_async_gen_to_sync_gen` does a blocking `queue.Queue.put()` with no timeout; when the SSE consumer stops iterating (browser navigates away, network drop), the worker loop thread freezes and the microVM can't serve any request until idle timeout. Confirm the symptom against our runtime, then mitigate — either a timeout-bounded put with a disconnect check, or a defensive monkey-patch until an upstream fix lands. Track the issue for an official fix.
- **Subtracts**: no — defensive; the failure mode is a silent 78s+ stall that masquerades as a cold-start or quota problem.
